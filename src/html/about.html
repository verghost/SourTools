About This Web Extension:

Spider: A simple web spider that currently has the following modules

  • DOM-XSS: Detects (potential) DOM-based XSS using the methodology outlined here
  • Mailman: Searches for email addresses using regex; additional info here
  • Dircheck: Searches for indexable directories (directory listings) by checking for the classic \"Index of\" title tag.
  • FileFinder: Finds files by filtering file extensions (i.e. doc, cpp, exe, zip, etc...)
  • MediaFinder: Finds media by filtering file extensions (i.e. mp3, mp4, jpg, png, etc...)
  • KeyFinder: Finds API keys and other secrets by using a list of regex rules as well as entropy to find random-looking data.

Robobust: A tool built to test the program (It's pretty useless). It requests paths found in a website's robots.txt file.

Dirbuster: Bruteforces directories and files names on web servers.

  • Lists should be formatted with # denoting a comment and each directory/filename on its own separate line.
  • Each path in the list is normalized to include a preceding slash (not a trailing one, although those can be explicitly included).
  • Example Dirbuster list:
    1. /root
    2. #this is a comment
    3. /admin
    4. /cgi-bin/

Custom Fetch: Fetches resources specified by the user by requesting a url, then isolating a target from the result.

  • Resources can be added and toggled on/off in the settings.
  • The syntax for targets is: NAME:INDEX{REGEX}.
    REGEX is matched to a result and the first match is chosen.
    NAME and INDEX have different purposes based on which response type is specified:
    • HTML: Uses PARSED_DOCUMENT.querySelectorAll(NAME)[INDEX] to choose a target, so the standard query selector syntax (i.e. #ID, .CLASSNAME, TAGNAME) should be used. Default value of INDEX is 0.
    • JSON: Target is selected by PARSED_JSON_OBJECT[NAME]. When PARSED_JSON_OBJECT[NAME] is an array and INDEX is supplied, we can specify further is used to further specify. In the case where INDEX is not specified, arrays and groups will be stringified.
      • NAME: Used to identify the target. The purpose of this part is different depending on which response type is specified.
      • INDEX: Default value is 0.
  • Replacements can be specified for target and url fields to insert info based on where the web tool is run:
    • HOST;N = current host (i.e. some.website.com)
    • HOST;WP = current host with protocol (i.e. https://some.website.com)
    • HOST;IP = current ip address
    • HOST;URL = current URL (i.e. https://some.website.com/search?q=somesearch)
    • HOST;D = current domain name (i.e. website.com)
    For url only: INPUT;TYPE causes input to be accepted (TYPE can be "text" or "num").

Tutils (Text Utils)

  • Remove Spaces, Reverse string
  • URI/URL Encode/Decode
  • BASE64 Encode/Decode UTF-16
  • ROT13 Encode/Decode
  • SHA-1, SHA-256, SHA-384, SHA-512
  • MD5
  • Beautify JS and CSS
  • XSS and SQLi string encoding from Firefox Hackbar
